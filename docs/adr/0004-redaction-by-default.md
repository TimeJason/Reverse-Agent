# ADR 0004：默认脱敏

## 状态

已接受，2026-06-09。

## 背景

HTTP、浏览器和日志证据可能包含令牌、Cookie、密码、密钥、个人信息和业务机密。平台会把结果暴露给 MCP 客户端、AI 模型、日志和导出文件；任一默认开放的路径都可能造成不可逆的数据泄漏。

## 决策

采用 Redaction by default：

- 原始证据可以按策略保存在本地，但不是常规读取接口的默认视图。
- MCP、AI、常规查询、日志、错误信息和导出默认使用脱敏值。
- 默认规则至少覆盖 `Authorization`、`Cookie`、`Set-Cookie`、令牌、密码、API key、私钥、会话标识和可识别个人信息。
- 字段名、值模式、内容类型和用户规则共同参与脱敏。
- 脱敏结果必须记录状态和被处理的字段路径，便于审计规则效果。
- raw access 必须显式请求、通过策略检查并写入审计日志。
- LLM 不得接收未脱敏证据，除非未来引入独立、明确授权且可审计的高级策略。
- 日志和审计元数据本身不得复制 raw secret。

## 影响

正面影响：

- 降低误把凭据或个人数据发送给模型、终端日志和导出文件的风险。
- 安全行为由默认值保证，而不是依赖每个调用者记得开启选项。
- 审计记录可以说明何时、为何访问了原始数据。

代价：

- 过度脱敏可能降低分析质量，需要安全的规则调试与例外机制。
- 必须维护检测规则、占位符稳定性和泄漏测试。
- 原始证据与脱敏视图的引用关系会增加存储复杂度。

## 替代方案

- **由调用者选择是否脱敏**：容易遗漏，无法形成一致安全边界。
- **不保存原始值**：风险最低，但会损失合法的本地重分析和调试能力。
- **仅在发送给 LLM 前脱敏**：无法覆盖 MCP、日志、错误和导出等其他泄漏路径。

## 后续工作

- 定义默认规则集、稳定占位符和用户覆盖顺序。
- 建立 secret leakage、日志泄漏和导出泄漏测试。
- 为 raw access 定义权限、确认和审计事件。
